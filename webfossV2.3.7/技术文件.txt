技 术 文 档
Webfoss网络科技项目

1、一些要求：
	-> 1、必须在最新版本的工程文件的基础写自己的代码
	-> 2、不要随意使用其它框架和插件
	-> 3、不要随意创建新文件，只在自己负责的页面的对应的文件进行操作，如：xxx.html、css/xxx.less、js/xxx.js、img/xxx.img 等文件。
	-> 3、创建一个名为“WebfossProject”的文件夹，文件中创建两个名为syVersions和myVersions的文件夹，用来存放下载来的综合版的历史版本和个人历史版本
	-> 4、每次有什么修改，有必要的请在工作日志中添加说明
	-> 5、工作结束，在工程文件末尾加上自己的版本号，例：webfossV2.1.5_ShenTaoV1.3
	-> 6、第次工作结束后请及时把项目文件发给汇总人汇总

2、尽量使用公共模块:
	-> 1、把功能模块（你封装的函数）放在js/functions.js中，加上函数使用说明
	-> 2、在工作日志中加上说明；

3、多看看reset.css，学会使用reset中的自定义类；

4、版本控制流程：
	工作开始..
	-> 1、下载最新版本的项目文件到WebfossProject中(并复制一个到syVersions的历史版本文件夹中)，下载方法如下：
		1.在WebfossProject中打开 Git Bush
		2.输入：git clone https://git.oschina.net/cntower/webfoss.git
	-> 3.下载完成
	-> 2、打开最新版本的项目文件，并在其基础上开始编辑
	-> 3、时时在"工作日志.txt"添加修改说明，请1、2、3、4...写清楚 
	-> 4、在编辑过的工程文件后面加上自己的版本号，如：webfossV2.1.4_ShenTaoV1.3
	-> 5、提交给汇总人汇总
	..工作结束!

5、汇总工作说明：
	-> 1、创建一个版本控件夹，包含：versions、王惠勋、李海洋、祈俊、崔丹丹、沈涛 的子文件夹和一个最新版本的工程文件，
	      子文件夹，用来存放综合和个人的历史版本。
	-> 2、把接收到的文件放在和最新版工程文件同一个目录下，并复制一份到对应成员的历史版本文件夹中。
	-> 3、把收到的文件合并到原来最新版本的文件中。
	-> 4、删除接收到的文件，并把合并后生成的最新版本的文件复制一份到versions文件夹中。
	-> 5、给每个成员分发最新版本的文件。
